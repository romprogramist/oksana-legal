# Дизайн: полное удаление онлайн-оплаты (эквайринга «Точка»)

**Дата:** 2026-06-07
**Статус:** утверждён, ожидает ревью спеки
**Откатывает:** `2026-04-26-tochka-payment-design.md`

## Цель

Убрать с сайта любую возможность онлайн-оплаты. Интернет-эквайринга («Точка»)
быть не должно. Услуги остаются платными, но оплата происходит офлайн по
договору — цены на лендинге сохраняются как справочная информация.

## Объём (вариант А: только слой эквайринга)

Удаляется **механизм онлайн-приёма денег** целиком. Блок цен и юридические
страницы (`/offer`, `/requisites`, `/refund`) **сохраняются**.

### 1. Удалить файлы и папки

| Путь | Назначение |
|---|---|
| `src/app/payment/` | страница онлайн-оплаты |
| `src/app/payment-methods/` | страница «Способы оплаты» |
| `src/app/api/payment/` (`init`, `notify`, `status`) | эндпоинты эквайринга |
| `src/app/api/admin/payments/route.ts` | API списка платежей в админке |
| `src/app/admin/payments/page.tsx` | раздел «Платежи» в админке |
| `src/lib/tochka.ts` | клиент эквайринга «Точки» |

### 2. Поправить файлы

- **`src/lib/constants.ts`** — убрать из `NAV_LINKS` пункт
  `{ href: "/payment-methods", label: "Оплата" }` и неиспользуемую после
  удаления страницы константу `PAYMENT_METHODS`.
- **`src/components/Footer.tsx`** — убрать `<li>` «Способы оплаты»
  (ссылка на `/payment-methods`).
- **`src/components/admin/AdminLayout.tsx`** — убрать пункт меню
  `{ href: "/admin/payments", label: "Платежи" }`.
- **`src/lib/rate-limit.ts`** — убрать `PAYMENT_WINDOW_MS`,
  `PAYMENT_MAX_PER_WINDOW`, `paymentStore`, `checkPaymentRateLimit`,
  `resetPaymentRateLimit`. Контактный rate-limit не трогать.
- **`src/app/sitemap.ts`** — убрать запись для `/payment`.
- **`src/app/requisites/page.tsx`** — убрать ставшую битой ссылку на
  `/payment-methods` (стр. ~143).
- **`.env`** (и `.env.example`, если есть) — удалить переменные `TOCHKA_*`
  (`TOCHKA_JWT_TOKEN`, `TOCHKA_API_URL`, `TOCHKA_PUBLIC_KEY_URL` и прочие
  `TOCHKA_`-ключи).

### 3. База данных

- **`prisma/schema.prisma`** — удалить `model Payment` и `enum PaymentStatus`.
- Сгенерировать миграцию, **дропающую таблицу `payments`** и тип
  `PaymentStatus`. Бэкап данных не делаем — записи признаны тестовыми
  (подтверждено заказчиком).

## Осознанно сохраняется

- **Блок «Стоимость услуг»** (`src/components/PricingSection.tsx`) и цены.
- **Юр-страницы** `/offer`, `/requisites`, `/refund`. Их текст может ссылаться
  на онлайн-оплату — корректировка формулировок это отдельная контент-задача
  для заказчика, **вне объёма данной правки кода**.
- Ссылки из `PricingSection` на `/offer` и `/refund` — остаются рабочими.

## Критерии готовности (verification)

1. `npm run build` проходит без ошибок (нет битых импортов после удаления
   `tochka.ts`, `PAYMENT_METHODS`, payment-функций rate-limit).
2. `grep -rE '/payment|/payment-methods|tochka|TOCHKA|checkPaymentRateLimit'
   src .env` не находит живых ссылок (допустимы только упоминания внутри
   оставленных юр-страниц, если они есть в тексте — не код).
3. Открытие `/payment` и `/payment-methods` возвращает 404.
4. В шапке сайта нет пункта «Оплата»; в админке нет раздела «Платежи».
5. Миграция применяется чисто; таблицы `payments` в БД нет.
6. Лендинг (`/`), админка и контактная форма работают как прежде.

## Риски

- **Миграция необратима** (drop table) — данные платежей теряются. Принято.
- Возможны прод-ссылки/закладки на `/payment*` — после удаления отдадут 404.
  Редирект не требуется (страницы были служебными).
